# Faceless-видео — кит для запуска (LawCheck)

_Готовый набор для коротких вертикальных видео без лица. Составлен 2026-06-26.
Цель: охватный верх воронки → переходы на бесплатную проверку. Формат — демо
продукта и польза, а не «говорящая голова»._

> Площадки: **VK Клипы, YouTube Shorts, Дзен, RUTUBE**. НЕ Instagram Reels —
> Meta запрещена в РФ, рекламы/охватов там нет. Один ролик → кросспост во все.
> Вертикаль 9:16, 15–45 сек, хук в первые 2 секунды, субтитры обязательны
> (смотрят без звука).

---

## 1. Принципы, по которым ролик «залетает»

- **Хук на деньгах/страхе в первые 2 секунды.** «За эту галочку — 700 000 ₽».
- **Конкретика, а не теория.** Реальный экран, реальная сумма, реальная норма.
- **Один ролик — одна мысль.** Не пытаться рассказать весь 152-ФЗ.
- **Субтитры всегда** (80% смотрят без звука).
- **CTA в конце и в описании:** «Проверь свой сайт бесплатно — ссылка в профиле».
- **Демо = контент.** Лучший формат для нас — запись экрана проверки сайта:
  это одновременно польза и демонстрация продукта.

## 2. Три рабочих формата

| Формат | Что это | Сложность |
|--------|---------|-----------|
| 🖥️ Скрин-разбор | запись экрана: проверяем реальный сайт, находки красным | низкая |
| 🗂️ «Штраф дня» | карточки-слайды с кейсом штрафа + озвучка | очень низкая |
| 📋 «N ошибок в…» | слайдер-список ошибок с примерами | очень низкая |

Начать с «Штраф дня» и «N ошибок» (делаются за 15–20 мин в слайд-редакторе),
скрин-разбор подключить, когда отладится процесс.

## 3. Производство без лица

1. **Картинка:**
   - Скрин-разбор → запись экрана (OBS Studio бесплатно; встроенная запись macOS/Win).
     Записывать в окне 9:16 или кадрировать при монтаже.
   - Карточки/слайдер → собрать слайды в Canva-аналоге (Supa, «ВК Дизайн»,
     Templated) и анимировать при монтаже.
2. **Голос (без лица):**
   - Свой голос за кадром (микрофон телефона ок) — самый «живой» вариант.
   - Или нейроозвучка: Yandex SpeechKit, российские TTS-сервисы, нейросети
     озвучки текста. Брать естественный голос, не «робота».
3. **Монтаж и субтитры:** мобильные редакторы (CapCut/InShot/VN/Mojo) или
   десктоп. Автосубтитры + ручная вычитка. Крупный шрифт, контраст.
4. **Конвейер:** один сценарий → озвучка → монтаж вертикали → автонарезка/экспорт
   → загрузка в VK Клипы / Shorts / Дзен / RUTUBE с одним описанием и хештегами.

> Ритм: 3–4 ролика в неделю минимум 4 недели, прежде чем делать выводы по охватам.

## 4. Готовые сценарии (раскадровки)

Формат: время · [на экране] · «озвучка/субтитр».

### Сценарий A — Скрин-разбор «проверка сайта за 60 секунд» (флагман)
```
0–2с  [крупно: поле ввода сайта на lawchek.ru, печатается адрес]
      «Берём обычный сайт магазина и проверяем за минуту.»
2–6с  [идёт проверка, появляется отчёт, критичные пункты красным]
      «И сразу три нарушения, за которые штрафуют.»
6–14с [навести/зум на пункт 1 → 2 → 3]
      «Нет согласия у формы. Политика с чужими реквизитами.
       Трекер грузится до согласия на cookie.»
14–20с[крупно: блок «суммарный риск штрафа — до … ₽»]
      «Суммарный риск — сотни тысяч рублей. И это видно снаружи,
       без доступа к сайту.»
20–24с[экран: главная lawchek.ru]
      «Проверь свой так же — бесплатно, ссылка в профиле.»
```
> Брать НЕ реальный чужой бренд (репутация/претензии) — демо-сайт или
> обезличенный пример.

### Сценарий B — «Штраф дня» (карточки)
```
0–2с  [чёрный фон, крупно: «700 000 ₽»]
      «Семьсот тысяч рублей — за одну галочку на сайте.»
2–8с  [карточка: форма заявки без согласия, обведено красным]
      «Самое частое нарушение: форма есть, а согласия на обработку
       персональных данных рядом нет.»
8–14с [карточка: ст. 13.11 ч. 2 КоАП]
      «Это статья 13.11 КоАП. И проверить это РКН может, просто
       открыв ваш сайт.»
14–18с[карточка с CTA]
      «Проверь свой сайт за 60 секунд — ссылка в профиле.»
```

### Сценарий C — «3 ошибки в политике конфиденциальности» (слайдер)
```
0–2с  [крупно: «Ваша политика конфиденциальности — фикция?»]
      «Три ошибки, из-за которых политика не защищает.»
2–7с  [слайд 1]
      «Первое: чужие реквизиты. Скачали шаблон вместе с чужим ИНН.»
7–12с [слайд 2]
      «Второе: политика не совпадает с сайтом. Есть оплата и запись,
       а в политике — ни слова.»
12–17с[слайд 3]
      «Третье: нет ссылки или ссылка битая. Документ есть — а доступа нет.»
17–21с[CTA]
      «Проверь свою политику бесплатно — ссылка в профиле.»
```

### Сценарий D — newsjacking (шаблон под инфоповод)
```
0–2с  [крупно: суть новости — «Штрафы за утечки выросли»]
      «[Что произошло — одним предложением.]»
2–10с [пояснение + что это значит для бизнеса]
      «[Вывод для владельца сайта.]»
10–15с[CTA]
      «Проверь, как у тебя с этим — ссылка в профиле.»
```
> Публиковать в день инфоповода (ужесточение штрафов, громкая утечка, новость РКН).

## 5. Библиотека хуков (первые 2 секунды)

- «За эту галочку — 700 000 ₽.»
- «Ваш сайт нарушает закон, и вы об этом не знаете.»
- «РКН не нужен выезд, чтобы вас оштрафовать.»
- «Проверил сайт за минуту — нашёл три нарушения.»
- «Эта ошибка есть у 8 сайтов из 10.»
- «Скачали шаблон политики? У вас проблема.»

## 6. Описание и хештеги (под каждый ролик)

```
Проверь свой сайт на нарушения 152-ФЗ бесплатно за 60 секунд → lawchek.ru

#152фз #персональныеданные #роскомнадзор #бизнес #интернетмагазин #штрафы
```
- В VK Клипы/Shorts ссылку — в профиль/описание канала (кликабельные ссылки
  в подписи ролика ограничены).
- UTM на переходы из профиля: `?utm_source=shorts` / `vkclips` / `dzen` / `rutube`.

## 7. Метрики

- Удержание (досматриваемость) — главный сигнал площадке; растить за счёт хука.
- Охваты и переходы в профиль → на сайт (UTM).
- Конверсия переходов в `scan_submit` (видно в Метрике).
- Через 4 недели: оставить форматы и хуки с лучшим удержанием/переходами.

## 8. Связь с другими каналами

- Те же темы, что в блоге (8 статей) и Telegram-ките — переупаковка контента.
- Ролик → пост в Telegram → статья в блоге: одна тема в трёх форматах.
