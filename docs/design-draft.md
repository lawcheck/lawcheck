# LawCheck — design draft

Автоматическая проверка сайтов на соответствие законодательству РФ (152-ФЗ, ЗОЗПП, 38-ФЗ, 436-ФЗ, 149-ФЗ, ПП №2463).
Один HTML-файл, три экрана, hash-роутинг (`#landing`, `#report`, `#pricing`). React + Tailwind, lucide-style иконки.

---

## 1. Экраны / страницы

| Экран | Hash | Назначение |
|---|---|---|
| **Главная** | `#landing` | Запуск проверки, объяснение что попадает в отчёт, соцдоказательство |
| **Отчёт** | `#report` | Результат проверки одного сайта: светофор + секции с findings |
| **Тарифы** | `#pricing` | Free / Pro / Business, сравнение, FAQ |

Общий каркас: sticky top-nav + footer на всех экранах.

### Top-nav
- Лого LawCheck + бейдж `beta`
- Пункты: Главная · Пример отчёта · Тарифы (активный подсвечен)
- Справа: GitHub · Документация · Войти · кнопка **«Проверить сайт»** (тёмная)

### Footer
- Колонки: Продукт / Законы / Компания
- Нижняя строка: © 2026, ОГРН, версия сборки и база правил (моно)

## 2. Hero (Главная)

- **Бейдж над заголовком:** `30 проверок · 6 законов · обновлено 14 мая 2026` (с зелёной точкой)
- **Заголовок:** «Проверьте сайт на соответствие **152-ФЗ** за 60 секунд.» (152-ФЗ — синим)
- **Подзаголовок:** обход сайта → находим нарушения по 152-ФЗ, ЗОЗПП, рекламе, защите детей; «не галочки, а конкретные цитаты со страниц с рекомендациями»
- **Trust-строка:** без регистрации · отчёт публичный по ссылке
- **Визуал / CTA (правая колонка):** карточка-форма запуска
  - Поле URL (префикс `https://`, моно)
  - Глубина проверки (число страниц) + Регион проверки (select: РФ / РФ + GDPR)
  - Кнопка **«Проверить сайт»** → `#report`
  - Сноска: «не храним содержимое страниц после генерации отчёта»
- Под формой — 3 мини-карточки «что проверяем»: Политика ПДн · Формы и реквизиты · Cookies и реклама

### Отчёт (`/report/{id}`)

| # | Блок | Цель | Ключевой контент |
|---|---|---|---|
| 1 | Хедер | Идентификация скана | Breadcrumb, URL `mystore.ru` (моно, крупно) + пилюля HTTPS·200; серый штамп: дата, «обработано страниц: 10 из 47», длительность, версия |
| 2 | Summary-светофор | Быстрая оценка | 4 critical · 3 warning · 2 info · 18 ok (приглушённые цвета) + прогресс-бар соответствия 67% |
| 3 | Running-плашка (опц.) | Состояние скана | Спиннер + «Идёт сканирование… обновится через 3 сек» (включается чекбоксом-демо в хедере) |
| 4 | Оглавление (sticky слева) | Навигация | Список 8 секций с маркером проблем/галочкой |
| 5 | Секции findings | Детали | 8 блоков: Политика ПДн · Формы и согласия · Cookies и трекеры · Реквизиты владельца · Реестр РКН · ЗОЗПП и Правила продажи · ФЗ «О рекламе» · Защита детей (436-ФЗ) |
| 6 | Footer отчёта | Апсейл | «Проверять раз в неделю?» → Включить мониторинг (`#pricing`) |

**Карточка finding:** левая цветная полоса по severity, иконка, заголовок, норма закона + код правила (`PDN-04`), пилюля severity, оценка штрафа, блок «where», цитата со страницы (моно), «почему нарушение», блок «как исправить» с готовым текстом, ссылка на статью закона.
**OK-findings** свёрнуты в раскрывающийся блок («N проверок прошли»); critical/warning/info — раскрыты всегда.

## 5. Тон и стиль

- Обращение на **«вы»**, экспертно и по-деловому, без панибратства.
- Продуктовый, а не маркетинговый: минимум воды, максимум сути и конкретики (цитаты, номера статей, коды правил, суммы штрафов).
- Не алармовый: severity-цвета приглушены, формулировки спокойные («нужно уточнить», а не «СРОЧНО!»).
- Технические значения везде моноширинным (URL, коды правил, версии) — подчёркивает «инженерный» характер сервиса.

## 7. Не решено / заглушки

- **Бэкенд проверки** — данные отчёта захардкожены (1 демо-сайт `mystore.ru`); нет реального обхода, очереди, реал-тайм обновления.
- **Состояние running** — статичная плашка, включается демо-чекбоксом; нет авто-рефреша / polling.
- **Кнопки без переходов:** поделиться, JSON, Документация, GitHub, Войти, Business → «Связаться» (нет форм/модалок).
- **Авторизация и личный кабинет** — отсутствуют (кнопка «Войти» — заглушка).
- **Подтверждение владения сайтом** (DNS TXT / meta) — упомянуто в FAQ, UI нет.
- **Оплата** — кнопки тарифов ведут на лендинг, нет чек-аута / платёжного флоу.
- **Мобильная версия отчёта** — оглавление скрыто на мобильном; не проверена плотность длинных findings на узких экранах.
- **Контент-заглушки:** суммы штрафов, число unit-тестов (153), статистика РКН («87 предписаний») — нужны реальные источники перед публикацией.
- **Ссылки на статьи законов** — все ведут на `#`, нет реальной базы норм.
