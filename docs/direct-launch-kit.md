## 2. Настройки кампании

- **Площадки:** только Поиск (РСЯ выключить).
- **География:** Россия. (При желании исключить — настроить позже по статистике.)
- **Расписание:** круглосуточно для старта; скорректировать по данным.
- **Счётчик Метрики:** 109808358.
- **Ключевая цель:** `scan_submit` (запуск проверки). Вторичная — `pay_click`.
- **Стратегия:**
  - Первые ~2 недели (нет статистики конверсий): «Максимум кликов» с
    ограничением средней цены клика (например, 60–90 ₽), чтобы собрать данные.
  - Когда накопится 10+ конверсий/нед — переключить на «Максимум конверсий»
    по цели `scan_submit` с недельным бюджетом.
- **Автотаргетинг:** на старте выключить (или режим «целевые запросы»), иначе
  открутит бюджет на нерелевантном.
- **Размечать ссылки для Метрики (yclid):** включить — связь Директ↔Метрика
  без ручных UTM. Дополнительно UTM-шаблон (ниже) для сквозной аналитики.

### Бюджет
- Старт: **500–1 000 ₽/день** (≈15–30 тыс ₽/мес).
- Клик в B2B-комплаенсе ориентировочно 40–150 ₽ — ждать десятки кликов в день.
- Решение по каналу принимать по **CPL** (стоимость `scan_submit`) и далее по
  CR `scan_submit → pay_click`, а не по цене клика.

### UTM-шаблон (в поле «Ссылка» или шаблон кампании)
```
?utm_source=yandex&utm_medium=cpc&utm_campaign=search_152fz&utm_term={keyword}&utm_content={ad_id}
```

## 6. После запуска — что смотреть

1. **CPL** = расход / число `scan_submit`. Главная метрика канала.
2. **CR `scan_submit` → `pay_click`** — узкое место воронки важнее цены клика.
3. **Поисковые запросы** (отчёт «Поисковые запросы») — раз в 2–3 дня чистить
   мусор в минус-слова.
4. Через 2 недели: отключить группы/фразы с дорогими лидами, бюджет — в
   рабочие. При достатке конверсий — сменить стратегию на «Максимум конверсий».
