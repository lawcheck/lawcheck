Политика обработки персональных данных

1. Общие положения

Настоящая Политика обработки персональных данных (далее — Политика) принята {{ operator.name }}{% if operator.inn %} (ИНН {{ operator.inn }}{% if operator.ogrnip %}, ОГРНИП {{ operator.ogrnip }}{% endif %}){% endif %} (далее — Оператор) в соответствии с Федеральным законом от 27.07.2006 № 152-ФЗ «О персональных данных» и определяет порядок обработки персональных данных пользователей сервиса LawCheck (lawchek.ru).

2. Цели обработки персональных данных

Обработка персональных данных осуществляется в целях:

  • предоставления сервиса автоматической проверки сайтов и формирования отчётов;
  • обеспечения работоспособности и безопасности сервиса, предотвращения злоупотреблений;
  • ответов на обращения пользователей и оказания поддержки;
  • исполнения договоров на платных тарифах, включая подготовку юридических заключений.

3. Правовые основания обработки

Правовым основанием обработки персональных данных являются: согласие субъекта персональных данных (ст. 6 ч. 1 п. 1 152-ФЗ); необходимость исполнения договора, стороной которого является субъект (ст. 6 ч. 1 п. 5 152-ФЗ); законные интересы Оператора по обеспечению работы и защиты сервиса.

4. Категории субъектов персональных данных

Категории субъектов персональных данных, чьи данные обрабатывает Оператор:

  • посетители и пользователи сайта lawchek.ru;
  • лица, направившие обращение Оператору;
  • клиенты платных тарифов.

5. Категории персональных данных

Перечень обрабатываемых персональных данных:

  • технические данные посетителей: IP-адрес, сведения о браузере (user-agent), дата и время обращения — фиксируются в серверных журналах;
  • адрес электронной почты и содержание обращения — при направлении обращения;
  • данные, необходимые для исполнения договора на платных тарифах (имя, e-mail, реквизиты для документов).

Адреса проверяемых сайтов, вводимые в форму проверки, сами по себе персональными данными не являются. Сервис не использует сторонние системы аналитики и рекламные трекеры; cookie-файлы для идентификации посетителей не применяются. Содержимое страниц проверяемых сайтов после формирования отчёта не хранится — в отчёте сохраняются только короткие цитаты-доказательства из общедоступных страниц.

6. Способы обработки

Способы обработки персональных данных: обработка осуществляется с использованием средств автоматизации (сбор, запись, систематизация, накопление, хранение, извлечение, использование, удаление, уничтожение).

7. Сроки обработки и хранения

Сроки обработки персональных данных определяются целями обработки: серверные журналы хранятся не более 90 дней; данные обращений — до завершения переписки и в течение 1 года после; данные клиентов платных тарифов — в течение срока договора и сроков, установленных законодательством о бухгалтерском учёте.

8. Порядок уничтожения персональных данных

По достижении целей обработки персональные данные уничтожаются. Уничтожение персональных данных осуществляется способом, исключающим их восстановление: автоматическая ротация журналов, удаление записей из базы данных по истечении сроков хранения либо по требованию субъекта.

9. Права субъекта персональных данных

Субъект персональных данных имеет право:

  • получать сведения об обработке своих персональных данных;
  • требовать уточнения, блокирования или уничтожения данных, если они неполны, устарели, неточны или обрабатываются с нарушением закона;
  • отозвать согласие на обработку;
  • обжаловать действия Оператора в Роскомнадзор или в суд.

10. Меры по обеспечению безопасности персональных данных

Оператор принимает следующие меры по обеспечению безопасности персональных данных: соединение с сайтом защищено протоколом HTTPS (TLS); доступ к данным ограничен; серверная инфраструктура размещена в дата-центре на территории Российской Федерации; применяются организационные и технические меры, предусмотренные ст. 19 152-ФЗ, исключающие несанкционированный доступ к персональным данным.

11. Трансграничная передача

Трансграничная передача персональных данных не осуществляется. Все данные обрабатываются и хранятся на серверах, расположенных на территории Российской Федерации; внешние ресурсы (шрифты, скрипты) загружаются с собственного домена сервиса.
